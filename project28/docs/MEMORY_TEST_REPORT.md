# 内存稳定性测试报告

## 测试概述

本文档描述如何验证优化后的实时股票监控看板在长时间运行下的内存稳定性。

## 测试目标

验证以下优化措施在连续1小时运行后：
1. **滚动窗口机制** - 内存使用保持恒定
2. **流式更新机制** - 无内存泄漏
3. **数据聚合机制** - 资源占用可控

## 验收标准

| 指标 | 优秀 (PASS) | 良好 (GOOD) | 失败 (FAIL) |
|-----|------------|-------------|-------------|
| 内存增长 | < 100 MB    | 100-500 MB  | > 2000 MB   |
| 峰值内存 | < 300 MB    | 300-800 MB  | > 2000 MB   |
| 增长率  | < 50 MB/h   | 50-200 MB/h | > 500 MB/h  |

## 测试环境要求

- Python 3.6+
- 依赖包: bokeh>=2.3.3, tornado>=6.1, websockets>=9.1, psutil>=5.8
- 操作系统: Windows/macOS/Linux

## 测试执行方法

### 方式1: 独立测试（不依赖WebSocket/Dashboard）

```bash

python run_memory_test.py --duration 60

# 完整1小时测试
python run_memory_test.py --duration 3600
```

### 方式2: 集成测试（需要WebSocket + Dashboard）

## 测试数据分析

### 如何分析CSV报告

测试结束后会生成CSV报告文件：
- 文件名: `memory_report_YYYYMMDD_HHMMSS.csv`
- 字段: timestamp, rss_mb, vms_mb, percent

```python
import pandas as pd

df = pd.read_csv('memory_report_*.csv')

print(f"采样点数: {len(df)}")
print(f"内存增长: {df['rss_mb'].iloc[-1] - df['rss_mb'].iloc[0]:.2f} MB")
print(f"峰值内存: {df['rss_mb'].max():.2f} MB")
print(f"平均增长率: {(df['rss_mb'].iloc[-1] - df['rss_mb'].iloc[0]) / (len(df) / 60):.2f} MB/小时")
```

### 内存增长趋势判定

**正常情况**（优化后）：
