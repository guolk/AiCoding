# 在线代码运行面板 - 核心模块实现说明

## 📋 项目概述

本项目是一个基于 React + Node.js 的全栈在线代码运行面板，提供安全的 JavaScript 代码执行环境。

---

```

│   │   ├── codeExecution.js          # 代码执行路由
```

## 🔧 后端核心模块

### 1. SafeCodeExecutor.js - 安全代码执行器

**文件位置**: `backend/utils/SafeCodeExecutor.js`

**核心职责**: 在隔离的沙箱环境中安全执行 JavaScript 代码

#### 1.1 初始化配置

```javascript
constructor(options = {}) {
  this.timeout = options.timeout || 5000;           // 执行超时时间（毫秒）
  this.memoryLimitMB = options.memoryLimitMB || 100; // 内存限制（MB）
  this.allowedBuiltins = [...];                       // 允许的内置对象
}
```

#### 1.2 沙箱配置

使用 `vm2` 库创建隔离执行环境：

```javascript
const vm = new NodeVM({
  timeout: this.timeout,      // 超时自动终止
  sandbox: {},                // 空沙箱，无预定义全局变量
  eval: false,                // 禁止 eval
  wasm: false,                // 禁止 WebAssembly
  allowAsync: true,           // 允许异步操作
  console: 'redirect',        // 重定向控制台输出
  require: {
    external: false,          // 禁止外部模块
    builtin: [],              // 禁止内置模块
    root: [],                 // 禁止 require 路径
  },
});
```

**安全设计要点**:
- ✅ **沙箱隔离**: 代码运行在独立的虚拟机中，无法访问 Node.js 核心模块
- ✅ **超时控制**: 默认 5 秒超时，防止无限循环阻塞服务器
- ✅ **禁止危险操作**: 禁用 `eval`、`new Function`、`require` 等
- ✅ **内存限制**: 通过配置限制内存使用

#### 1.3 代码清理 (sanitizeCode)

在执行前进行静态代码分析，检测危险模式：

```javascript
const dangerousPatterns = [
  /process\s*\./g,           // 禁止访问 process 对象
  /require\s*\(/g,           // 禁止 require 调用
  /global\s*\./g,            // 禁止 global 对象
  /module\s*\./g,            // 禁止 module 对象
  /__dirname/g,              // 禁止 __dirname
  /__filename/g,             // 禁止 __filename
  /eval\s*\(/g,              // 禁止 eval
  /new\s+Function/g,         // 禁止 new Function
  /setTimeout\s*\(/g,        // 禁止定时器
  /setInterval\s*\(/g,
  /child_process/g,          // 禁止子进程
  /fs\s*\./g,                // 禁止文件系统
  /http\s*\./g,              // 禁止网络操作
  /https\s*\./g,
  /net\s*\./g,
  /os\s*\./g,                // 禁止操作系统信息
  /vm\s*\./g,                // 禁止虚拟机操作
];
```

#### 1.4 执行结果格式

**成功执行返回**:
```javascript
{
  success: true,
  status: 'success',
  message: '代码执行成功',
  logs: [
    { type: 'log', content: 'Hello', timestamp: '2026-04-23T...' }
  ],
  result: '"返回值"',           // JSON 序列化后的结果
  executionTime: 15,            // 执行耗时（毫秒）
  startTime: 1234567890,
  endTime: 1234567890123
}
```

**失败执行返回**:
```javascript
{
  success: false,
  status: 'error',
  message: '代码执行失败',
  errorType: 'RuntimeError',
  logs: [...],
  errors: [
    { 
      type: 'runtime_error',
      message: 'ReferenceError: x is not defined',
      name: 'ReferenceError',
      stack: '...',
      timestamp: '...'
    }
  ],
  executionTime: 5
}
```

### 2. HistoryManager.js - 历史记录管理器

**文件位置**: `backend/utils/HistoryManager.js`

**核心职责**: 管理代码执行历史记录的持久化存储

#### 2.1 数据结构

```javascript
// 单条记录结构
{
  id: 'uuid-v4',               // 唯一标识
  code: 'console.log("test")', // 执行的代码
  result: 'undefined',          // 返回值
  logs: [...],                  // 控制台输出
  createdAt: 'ISO string',      // 创建时间
  updatedAt: 'ISO string'       // 更新时间
}
```

#### 2.2 核心方法

| 方法 | 描述 |
|------|------|
| `addRecord(code, result, logs)` | 添加新记录，自动限制数量 |
| `getHistory(limit)` | 获取历史记录列表 |
| `getRecordById(id)` | 根据 ID 获取单条记录 |
| `deleteRecord(id)` | 删除指定记录 |
| `clearAll()` | 清空所有记录 |

#### 2.3 存储机制

- 使用 JSON 文件本地存储 (`data/history.json`)
- 自动创建数据目录
- 限制最多保存 100 条记录（可配置）
- 采用队列方式：新记录插入头部，超出限制时移除最旧记录

### 3. API 路由模块

#### 3.1 codeExecution.js - 代码执行路由

**文件位置**: `backend/routes/codeExecution.js`

**端点**: `POST /api/execute`

**请求体**:
```json
{
  "code": "console.log('Hello')"
}
```

**错误类型分类**:

| 错误类型 | HTTP 状态码 | 场景 |
|----------|-------------|------|
| `validation_error` | 400 | 代码为空、代码过长 |
| `security_error` | 403 | 代码包含危险模式 |
| `server_error` | 500 | 服务器内部错误 |
| `RuntimeError` | 200 | 运行时错误（如未定义变量） |

**设计要点**:
- 运行时错误返回 200 状态码，但 `success: false`，便于前端区分
- 包含执行时间统计，便于性能监控
- 自动保存到历史记录

#### 3.2 history.js - 历史记录路由

**文件位置**: `backend/routes/history.js`

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/history` | 获取历史记录列表（支持 `?limit=20`） |
| GET | `/api/history/:id` | 获取单条记录详情 |
| DELETE | `/api/history/:id` | 删除单条记录 |
| DELETE | `/api/history` | 清空所有记录 |

## 🎨 前端核心模块

### 1. App.jsx - 主应用组件

**文件位置**: `frontend/src/App.jsx`

#### 1.1 状态管理

```javascript
const [code, setCode] = useState('...');        // 代码输入
const [logs, setLogs] = useState([]);            // 控制台日志
const [result, setResult] = useState(null);      // 返回值
const [isRunning, setIsRunning] = useState(false); // 运行状态
const [executionStatus, setExecutionStatus] = useState(null); // 执行状态
const [history, setHistory] = useState([]);      // 历史记录列表
const [selectedHistoryId, setSelectedHistoryId] = useState(null);
const [theme, setTheme] = useState('light');     // 主题状态
```

#### 1.2 核心函数

**runCode() - 执行代码**:
```javascript
const runCode = async () => {
  setIsRunning(true);
  setLogs([]);
  setResult(null);
  setExecutionStatus(null);

  try {
    const response = await axios.post(`${API_BASE_URL}/execute`, { code });
    const data = response.data;
    
    // 更新执行状态（成功/失败）
    setExecutionStatus({
      success: data.success,
      status: data.status,
      message: data.message,
      errorType: data.errorType,
      detail: data.detail,
      executionTime: data.executionTime,
      totalExecutionTime: data.totalExecutionTime
    });

    // 更新日志和返回值
    if (data.logs) setLogs(data.logs);
    if (data.success && data.result) setResult(data.result);
    
    // 刷新历史记录
    if (data.historyId) {
      await loadHistory();
      setSelectedHistoryId(data.historyId);
    }
  } catch (error) {
    // 处理网络错误
    setExecutionStatus({
      success: false,
      status: 'network_error',
      message: '网络请求失败',
      errorType: 'NetworkError',
      detail: error.message,
      executionTime: null
    });
  } finally {
    setIsRunning(false);
  }
};
```

#### 1.3 状态图标映射

```javascript
const getStatusIcon = (status) => {
  if (!status) return null;
  if (status.success) return '✅';           // 成功
  if (status.status === 'validation_error') return '⚠️';   // 验证错误
  if (status.status === 'security_error') return '🔒';     // 安全错误
  if (status.status === 'server_error') return '💥';       // 服务器错误
  return '❌';                                              // 其他错误
};
```

## 🔐 安全机制详解

### 1. 多层防护

本系统采用多层安全防护机制：

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: 静态代码分析 (sanitizeCode)                    │
│  - 危险模式正则匹配                                        │
│  - 禁止访问 process/require/global 等                     │
├─────────────────────────────────────────────────────────┤
│  Layer 2: 沙箱隔离 (vm2 NodeVM)                          │
│  - 独立虚拟机环境                                          │
│  - 禁止 require 外部模块                                   │
│  - 超时控制                                                │
├─────────────────────────────────────────────────────────┤
│  Layer 3: 运行时限制                                      │
│  - 禁止 eval/new Function                                 │
│  - 禁止文件系统/网络操作                                    │
│  - 禁止子进程                                              │
└─────────────────────────────────────────────────────────┘
```

### 2. 禁止的危险操作

| 操作类型 | 示例 | 风险 |
|----------|------|------|
| 进程控制 | `process.exit()` | 终止服务器进程 |
| 模块加载 | `require('fs')` | 访问文件系统 |
| 全局对象 | `global.x = 'evil'` | 污染全局环境 |
| 动态执行 | `eval('evil code')` | 执行任意代码 |
| 定时器 | `setInterval(infinite)` | 资源耗尽 |
| 文件系统 | `fs.readFile('/etc/passwd')` | 敏感信息泄露 |
| 网络 | `http.get('malicious')` | 内网探测 |

### 3. 允许的操作

安全沙箱中允许的内置对象：
- `console` - 控制台输出
- `Array`, `Object`, `String`, `Number`, `Boolean` - 基础类型
- `Math`, `Date`, `JSON`, `RegExp` - 常用工具
- `Map`, `Set`, `WeakMap`, `WeakSet`, `Promise` - ES6+ 特性

## 🎯 状态码说明

| 状态类型 | success | status | HTTP 码 | 场景 |
|----------|---------|--------|---------|------|
| 执行成功 | true | `success` | 200 | 代码正常执行 |
| 运行时错误 | false | `error` | 200 | 如 `ReferenceError`、`TypeError` |
| 验证错误 | false | `validation_error` | 400 | 代码为空、过长 |
| 安全错误 | false | `security_error` | 403 | 包含危险模式 |
| 服务器错误 | false | `server_error` | 500 | 内部异常 |
| 网络错误 | false | `network_error` | - | 前端请求失败 |
