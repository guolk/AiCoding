# 乐队排练与作品管理平台

一个功能完整的在线乐队排练和作品管理系统，帮助乐队高效管理成员、曲目、排练、演出和资源。

## 功能模块

### 1. 仪表盘 (Dashboard)
- 全局数据概览：成员数、曲目数、排练记录、演出场次等
- 即将到来的排练和演出提醒
- 曲目状态统计（学习中/可演出/已归档）
- 日程日历视图
- 待解决的技术问题清单
- 成员出席率统计

### 2. 乐队管理 (Band Management)
**成员信息管理**
- 乐手个人信息：姓名、声部、联系方式
- 可排练时间设置
- 成员状态（活跃/非活跃）
- 出席率自动统计

**排练日程协调**
- 排练计划创建和管理
- 智能推荐最优排练时间（基于成员可用时间）
- 参与成员选择
- 排练地点和时间设置

**请假与缺席管理**
- 请假申请提交
- 请假审批流程
- 出席率自动计算

### 3. 作品库管理 (Song Library)
**曲目版本管理**
- 多版本管理：原版、编曲版、现场版、Demo版
- 曲目难度评级
- 时长记录

**乐谱管理**
- PDF乐谱上传
- 各声部分谱单独管理
- 乐谱标注功能

**演奏状态追踪**
- 学习中 / 可演出 / 已归档 三态管理
- 状态快速切换

### 4. 排练记录 (Rehearsal Records)
**排练记录**
- 参与人员记录
- 排练曲目记录
- 排练时长统计
- 重点练习片段标注

**技术问题管理**
- 问题标注（小节位置、问题描述）
- 负责人指派
- 解决状态追踪（待解决/已解决）

**录音关联**
- 排练录音上传
- 录音文件与排练记录关联

### 5. 演出管理 (Performance Management)
**演出场次记录**
- 演出地点、时间、观众数
- 演出曲目单管理
- 曲目排序

**演出准备清单**
- 装备清单管理
- 人员安排
- 彩排时间设置
- 任务完成状态追踪

**演出复盘**
- 总体评分
- 发挥亮点记录
- 改进建议记录

### 6. 资源共享 (Resource Sharing)
**设备借用管理**
- 设备信息登记
- 借用申请流程
- 借用审批
- 归还管理

**资源共享**
- 乐谱、录音、文档等资源上传
- 共享权限设置（指定成员/全员）
- 资源分类管理

## 技术栈

- **前端框架**: React 18 + TypeScript
- **UI 组件库**: Ant Design 5.x
- **路由**: React Router 6
- **日期处理**: Day.js
- **构建工具**: Vite
- **数据存储**: LocalStorage

## 快速开始

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000 查看应用

### 构建生产版本

```bash
npm run build
```

### 预览生产构建

```bash
npm run preview
```

## 项目结构

```
src/
├── types/              # TypeScript 类型定义
│   └── index.ts
├── utils/              # 工具函数
│   ├── storage.ts      # 本地存储封装
│   └── mockData.ts     # 示例数据
├── pages/              # 页面组件
│   ├── Dashboard/      # 仪表盘
│   ├── BandManagement/ # 乐队管理
│   ├── SongLibrary/    # 作品库
│   ├── RehearsalRecords/ # 排练记录
│   ├── PerformanceManagement/ # 演出管理
│   └── ResourceSharing/ # 资源共享
├── App.tsx             # 应用主组件
├── main.tsx            # 应用入口
└── index.css           # 全局样式
```

## 数据说明

应用首次加载时会自动生成示例数据，包括：
- 6 名乐队成员
- 5 首曲目
- 3 场排练计划
- 1 条排练记录
- 2 场演出
- 5 件设备

点击页面右上角的「重置示例数据」按钮可以随时恢复初始示例数据。

所有数据存储在浏览器的 LocalStorage 中，清除浏览器数据会导致数据丢失。
